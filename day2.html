Session 1

AI를 ‘비서’가 아니라 ‘초안 도구’로 이해하기

생성형 AI는 입력한 지시와 맥락을 바탕으로 글·요약·아이디어·표 등의 결과를 만들어 주는 도구입니다. 사람처럼 이해하거나 책임지는 존재가 아니며, 그럴듯하지만 틀린 내용을 만들 수도 있습니다. 따라서 AI는 초안을 빠르게 만드는 조력자, 최종 판단은 실무자라는 원칙으로 사용합니다.

AI (초안 조력자)
인간 (최종 판단자)
작성은 AI가 신속하게 진행하되,
최종 정합성 검증과 승인은 실무자가 진행합니다.

잘하는 일

요약, 분류, 초안 작성, 표현 바꾸기, 아이디어 확장, 체크리스트 만들기

주의할 일

최신 정보, 숫자·법·지침, 아동 개인 사례, 사실 여부는 반드시 원문과 담당자 확인

사람이 맡을 일

아동의 안전·권리·관계에 관한 판단, 사실 확인, 최종 승인, 현장 적용

기억할 문장　AI가 써준 문장은 ‘정답’이 아니라 ‘검토할 초안’입니다. 빠르게 쓰는 것보다 안전하게 확인하는 것이 중요합니다.
Session 2

왜 AI 답변을 그대로 믿으면 안 될까?

AI는 질문의 의도를 추측해 가장 그럴듯한 답변을 생성합니다. 답변의 말투가 자신 있어 보여도 사실이라는 뜻은 아닙니다. 존재하지 않는 출처·통계·사례를 제시하는 환각(hallucination)도 생길 수 있습니다.

AI 답변 생성
검증 필터링
안전한 최종 사용

안전한 4단계 업무 흐름

  1. 1. 초안　AI에게 정리·초안·대안을 요청
  2. 2. 대조　원문·기관 양식·공식 자료와 비교
  3. 3. 수정　틀린 내용, 과장, 차별적 표현, 개인정보를 제거
  4. 4. 승인　담당자가 최종 검토 후 사용

검증 질문 4개

  • 이 숫자와 날짜의 출처는 무엇인가?
  • 원문에 실제로 있는 내용인가?
  • 아동을 낙인찍거나 단정하는 표현은 없는가?
  • 현장 인력·예산·시간으로 실행 가능한가?
Session 3

GPT는 무슨 뜻일까?

GPT는 Generative Pre-trained Transformer의 앞글자를 딴 이름입니다. 어렵게 외우기보다 “미리 공부한 자료를 바탕으로, 앞뒤 맥락을 살피며, 다음 말을 만들어 내는 AI”라고 이해하면 됩니다.

03 · 트랜스포머
앞뒤 단어의 연결 맥락 분석

문장 안의 여러 단어가 서로 어떤 연결고리(관계)를 갖는지 살핍니다. 사용자의 프롬프트 앞뒤 맥락을 보고 다음에 들어갈 단어 확률을 예측해 문맥을 잇습니다.

기억할 문장　GPT는 ‘사람처럼 생각하는 사람’이 아니라, 학습한 언어 패턴과 현재 대화 맥락을 바탕으로 답변을 생성하는 모델입니다. 그래서 답변은 늘 확인이 필요합니다.
Session 4

NotebookLM이란?

NotebookLM은 내가 올린 자료를 중심으로 질문하고 정리하는 ‘자료 기반 AI 메모장’입니다. 일반 챗봇이 폭넓은 지식을 활용한다면, NotebookLM은 선택한 문서·PDF·웹페이지·유튜브 자막·Google 문서 등을 자료로 삼아 답변합니다.

STEP 1

자료 넣기

사업계획서, 운영지침, 공개된 통계, 회의자료 등 주제에 필요한 자료를 한 노트북에 모읍니다. 자료의 출처와 사용 권한을 먼저 확인합니다.

STEP 2

자료에 질문하기

“요약해줘”보다 “세 자료의 공통 목표와 차이점을 표로 정리해줘”처럼 대상·기준·형식을 함께 요청합니다. 답변에는 원문을 확인할 수 있는 인용이 표시됩니다.

STEP 3

결과물로 바꾸기

자료를 바탕으로 학습 가이드, 브리핑, 오디오 개요, 마인드맵, 퀴즈, 플래시카드, 표 등을 만들 수 있습니다. 기능과 사용량은 계정에 따라 다를 수 있습니다.

NotebookLM 자료 분석 시뮬레이션
나의 자료를 독점 파싱하여 가이드라인 도출

지역아동센터 활용 예

  • 공모사업 안내문 여러 개의 공통 제출서류 비교
  • 기관 운영지침에서 담당자가 확인할 절차 추출
  • 직원 교육자료를 오디오 개요·퀴즈로 바꾸기

개인정보·저작권 주의

아동의 사례기록·사진·상담내용은 실제 자료 그대로 올리지 않습니다. 기관이 승인한 자료만 사용하고, 공개·공유할 때는 원자료의 권리와 접근 범위를 확인합니다. NotebookLM도 AI이므로 의료·법률·보호 판단을 대신하지 않습니다.

쉽게 비유하면　NotebookLM은 ‘아무거나 대답하는 검색창’이 아니라, 내가 건넨 자료를 책상 위에 펼쳐 놓고 함께 찾아보는 ‘자료 정리 파트너’입니다. 그래도 인용문과 원문은 반드시 대조합니다.
Session 5

기본 프롬프트 6요소: 말 잘하기보다 일 잘 시키기

6요소를 모두 길게 쓸 필요는 없습니다. 목적과 맥락을 분명히 하고, 결과를 확인할 수 있게 요청하는 것이 핵심입니다. 역할 부여는 선택 사항이며, 역할을 준다고 전문성이 보장되지는 않습니다.

01 · 목적

무엇을 위해?

“지역아동센터 방학 프로그램 초안을 만들기 위해”

02 · 맥락

어떤 상황에서?

대상, 기간, 인력, 예산, 이미 확인한 사실

03 · 작업

무엇을 해줘?

요약, 비교, 초안, 질문 만들기처럼 동사로 요청

04 · 형식

어떤 모양으로?

표·목록·문단 수·글자 수·대상 독자를 지정

05 · 기준

무엇을 지켜?

근거 없는 숫자는 만들지 말고, 모르면 ‘확인 필요’로 표시

06 · 예시·검증

어떻게 확인해?

원하는 예시를 주고, 사실·추정·확인 필요를 구분하도록 요청

6요소 조립 기획 렌즈 (Puzzle Assembly)
목적/맥락
작업/형식
조각들이 맞물려 완벽한 업무 지시서 완성

프롬프트에도 넣지 않을 것

이름·연락처·주소·학교·얼굴 사진, 상담·보호·가정환경 기록, 건강·장애·진단 관련 정보, 사건·학대 의심 내용, 계정·비밀번호

실습용 데이터로 바꾸기

‘A아동’, ‘초등 고학년 8명’처럼 가명·범주·집계값을 사용하고, 불필요한 날짜·장소·관계 정보는 삭제합니다. 실제 자료는 기관 승인과 보안 설정을 먼저 확인합니다.

멈춤 신호　‘이 내용을 외부 서비스에 올려도 아동·보호자·기관에 설명할 수 있는가?’에 답하기 어렵다면 입력하지 않습니다.
목적: 지역아동센터 방학 프로그램 초안을 만들고 싶어. 맥락: 초등 고학년 8명, 4주, 실무자 2명, 주 2회, 예산 80만 원. 아래 내용은 비식별화한 가상 정보야. 작업: 프로그램 목표 2개와 회기별 활동표를 제안해줘. 형식: 표로 작성하고, 각 활동의 준비물·소요시간·담당자를 포함해줘. 기준: 근거 없는 통계나 비용은 만들지 말고 ‘확인 필요’라고 표시해줘. 아동을 낙인찍는 표현은 사용하지 마. 검증: 마지막에 실행 전 확인할 사항 5개를 덧붙여줘.
Session 6

지역아동센터 실무 프롬프트 4종

아래 예시는 실제 아동 정보가 아닌 가상 정보로 연습합니다. 결과를 복사하기 전에 원문·기관 양식·현장 가능성을 확인하세요.

가상 회의 메모를 붙여넣고 담당자·기한·확인 필요 사항을 분리합니다.

실습 프롬프트 1
아래 회의 메모를 바탕으로 실행 목록을 만들어줘. 1. 결정된 사항 / 2. 추가 논의가 필요한 사항 / 3. 담당자와 기한 / 4. 외부 확인이 필요한 사실로 나눠줘. 메모에 없는 담당자·기한·숫자는 만들지 말고 ‘미정’ 또는 ‘확인 필요’로 표시해줘. 표 형식으로 작성해줘. 회의 메모: [비식별화한 가상 메모를 붙여넣기]

대상·기간·인력·예산을 먼저 주고, 실행 가능성 질문까지 요청합니다.

실습 프롬프트 2
너는 지역아동센터 프로그램 기획을 돕는 조력자야. 아래 조건에 맞춰 3가지 프로그램 안을 제안해줘. 조건: 초등 고학년 8명 / 4주 / 주 2회 / 실무자 2명 / 예산 80만 원 / 목표는 협력과 자기표현 경험 확대 각 안에 목표·주요 활동·회기 수·준비물·예상 위험과 대응을 포함해줘. 아동을 진단하거나 낙인찍는 표현은 쓰지 말고, 확인되지 않은 효과를 단정하지 마. 마지막에 실무자가 선택하기 위해 추가로 확인할 질문 5개를 제시해줘.

NotebookLM에 기관 승인 자료나 공개 자료를 넣고, 인용과 원문 위치를 함께 확인합니다.

실습 프롬프트 3
내가 선택한 자료만 근거로 아래 항목을 정리해줘. 1. 문서에 명시된 핵심 문제 2. 사업 목표 3. 대상과 기간 4. 성과지표 5. 숫자·날짜·조건 각 항목마다 근거가 있는 쪽수 또는 문서 내 위치를 표시해줘. 문서에 없는 내용은 추론하지 말고 ‘문서에 없음’이라고 써줘. 마지막에 담당자가 원문으로 다시 확인할 항목을 5개 제안해줘. 첨부 자료: [기관 승인 자료 또는 공개 자료]

AI에게 ‘합격 보장’이나 ‘전문가 판정’을 요구하지 않고, 확인 가능한 개선점만 받습니다.

실습 프롬프트 4
아래 비식별화한 사업계획서 초안을 다음 기준으로 점검해줘. ① 목표와 활동이 연결되는가 ② 대상·기간·인력이 실행 가능한가 ③ 예산 산식에 빈칸이나 계산 오류가 있는가 ④ 사실·추정·확인 필요가 구분되는가 ⑤ 아동을 존중하는 표현인가 각 문제마다 ‘문제 문장 / 왜 확인이 필요한지 / 수정 방향’을 표로 작성해줘. 법·지침·통계는 출처를 확인하지 못하면 단정하지 말고 ‘공식 자료 확인 필요’라고 표시해줘. 초안: [비식별화한 초안]
교육 자료의 기준　UNESCO의 인간 중심·비판적 사고·윤리 원칙, NIST의 생성형 AI 위험관리 관점을 참고했습니다. NotebookLM 기능 설명은 Google 공식 도움말과 제공하신 NotebookLM 가이드를 대조해 정리했습니다. 서비스 기능과 개인정보 설정은 변경될 수 있으므로 실습 당일 공식 도움말과 기관 정책을 다시 확인합니다.
